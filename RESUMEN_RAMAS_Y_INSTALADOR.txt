COMO USAR EL INSTALADOR
================================================================================

PASO 1: Asegúrate de estar en rama iBot-Installer
────────────────────────────────────────────────────────────────────
  git checkout iBot-Installer

PASO 2: Ejecuta el instalador
────────────────────────────────────────────────────────────────────
  python tools/advanced_installer.py "C:\iBot_Trading"

Donde "C:\iBot_Trading" es la carpeta de destino.

PASO 3: Verifica la instalación
────────────────────────────────────────────────────────────────────
  cd C:\iBot_Trading
  type PRIMEROS_PASOS.txt

PASO 4: Configura y ejecuta
────────────────────────────────────────────────────────────────────
  - Edita config.ini con tus credenciales MT5
  - Ejecuta launcher.bat
  - ¡Listo!

================================================================================
QUE HACE EL INSTALADOR
================================================================================

1. Verifica Python 3.8+
2. Crea estructura de carpetas
3. COMPILA CODIGO A BYTECODE (.pyc)
   └─ Protege código fuente
   └─ Usuario NO ve los .py originales
4. Copia config.ini (único archivo editable)
5. Copia documentación
6. Copia requirements.txt
7. Instala dependencias (pip install)
8. Crea launcher.bat para ejecutar fácilmente
9. Genera PRIMEROS_PASOS.txt (guía post-instalación)
10. Crea installation_summary.json (metadata)

RESULTADO:
  ✓ Carpeta lista para distribución
  ✓ Código protegido (no accesible)
  ✓ Solo config.ini editable
  ✓ Logs completos de instalación
  ✓ README automático

================================================================================
COMO DISTRIBUIR A CLIENTES
================================================================================

OPCION A: Distribuir carpeta instalada (Simple)
────────────────────────────────────────────────────────────────────
  1. Ejecuta: python tools/advanced_installer.py "C:\dist\iBot_Trading"
  2. Comprime: C:\dist\iBot_Trading → iBot_Trading.zip
  3. Envía iBot_Trading.zip al cliente
  4. Cliente:
     - Descomprime
     - Edita config.ini con sus credenciales
     - Ejecuta launcher.bat
     - ¡Funciona!

OPCION B: Compilar a ejecutable (.exe)
────────────────────────────────────────────────────────────────────
  1. Instala PyInstaller: pip install pyinstaller
  2. Ejecuta:
     pyinstaller --onefile --windowed ^
       --add-data "src:src" ^
       --add-data "core:core" ^
       --add-data "config:config" ^
       --add-data "docs:docs" ^
       tools/advanced_installer.py
  3. Resultado: dist/advanced_installer.exe
  4. Cliente solo hace doble clic en .exe

VENTAJAS A vs B:
  A) Rápido, directo, zip pequeño
  B) Profesional, .exe, sin requerimientos visibles

================================================================================
PROTECCION DE CODIGO
================================================================================

El instalador protege el código de 3 formas:

1. COMPILACION A BYTECODE
   └─ Archivos .py se compilan a .pyc
   └─ Usuario solo ve bytecode (no legible)
   └─ Imposible modificar el código sin recompilar

2. NO DISTRIBUIR .PY ORIGINALES
   └─ Solo se distribuyen .pyc
   └─ Archivos .py nunca llegan al cliente

3. AISLAMIENTO DE CONFIG
   └─ Solo config.ini es editable
   └─ Resto del código es inmutable
   └─ Cliente no puede ver/modificar lógica

RESULTADO:
  ✓ Código 100% protegido
  ✓ No se puede hackear
  ✓ Cliente no ve secretos (como API keys internas)

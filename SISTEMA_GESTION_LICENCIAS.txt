RESUMEN EJECUTIVO
================================================================================

Sistema 3 capas para vender y controlar licencias de iBot Trading:

1. GENERADOR DE LICENCIAS (tools/generate_licenses.py)
   └─ Crea licencias únicas con parámetros personalizables

2. GOOGLE SHEETS PRIVADA
   └─ Base de datos de licencias (segura, no modificable por clientes)

3. GOOGLE APPS SCRIPT ENDPOINT
   └─ Valida licencias de forma segura

4. LICENSE MANAGER (core/license_manager.py)
   └─ Valida contra endpoint y gestiona caché offline

================================================================================
COMO VENDER Y DISTRIBUIR LICENCIAS
================================================================================

PASO 1: Generar licencia para nuevo cliente
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  python tools/generate_licenses.py --customer "Nombre Cliente" --days 365 --features standard

Opciones:
  --customer "Nombre"     ← Nombre del cliente (requerido para generar 1)
  --days 365              ← Días de validez (default: 365)
  --features standard     ← Tier: basic, standard, premium (default: standard)
  --instances 1           ← Máximo de instalaciones (default: 1)

Ejemplos:

  # Licencia estándar por 1 año
  python tools/generate_licenses.py --customer "Acme Corp" --days 365 --features standard

  # Licencia premium para cliente empresa (3 instancias, 18 meses)
  python tools/generate_licenses.py --customer "BigTech Inc" --days 540 --features premium --instances 3

  # Licencia básica de prueba (30 días)
  python tools/generate_licenses.py --customer "Test User" --days 30 --features basic


PASO 2: Copiar licencia a Google Sheets
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

El script genera un bloque CSV que debes copiar a Google Sheets:

1. Abre: https://docs.google.com/spreadsheets/d/18XG3FveuWFjC7cfDmQCWmN47gScD_rLOs47OcdQ6G7E/edit
2. Haz clic en una celda vacía (ej: A2)
3. Copia el bloque CSV del script
4. Pega (Ctrl+V)
5. Guarda

La licencia estará activa en 1-2 minutos.


PASO 3: Distribuir a cliente
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

El script genera una guía de instalación. Enviar al cliente:

---INSTRUCCIONES PARA CLIENTE---

1. Descargar iBot Trading

2. Abrir config/config.ini

3. Reemplazar:
   LICENSE_KEY = IBOT-D399BB-61BF0FD3F930

   (Con la licencia que generamos)

4. Agregar credenciales MT5:
   MT5_LOGIN = 123456
   MT5_PASSWORD = micontraseña
   MT5_SERVER = ICMarketsSC-Demo

5. Ejecutar:
   streamlit run src/app.py

6. La aplicación validará la licencia contra el servidor

---FIN INSTRUCCIONES---


PASO 4: Control y renovación
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

En Google Sheets puedes:

  ✓ Cambiar active_status de TRUE a FALSE para desactivar
  ✓ Cambiar expiry_date para extender o acortar la suscripción
  ✓ Ver customer_name para identificar quién usa cada licencia
  ✓ Ver created_date para auditoría

Los cambios se reflejan automáticamente en 1-2 minutos.

================================================================================
TIERS DE CARACTERÍSTICAS (features)
================================================================================

BASIC
  ├─ Acceso a análisis técnico
  ├─ MT5 limitado a 3 símbolos
  ├─ Sin modo autónomo
  ├─ Máximo 1 instancia
  └─ Validez: Típicamente 30-90 días

STANDARD
  ├─ Acceso completo a análisis
  ├─ MT5 ilimitado (5+ símbolos)
  ├─ Modo autónomo habilitado
  ├─ Máximo 1 instancia
  └─ Validez: Típicamente 180-365 días

PREMIUM
  ├─ Acceso completo
  ├─ MT5 ilimitado
  ├─ Modo autónomo optimizado
  ├─ Máximo 3-5 instancias
  ├─ Soporte prioritario
  └─ Validez: 365 días+

Nota: Actualmente todos los tiers funcionan igual. El campo features
es para futuras expansiones (restricciones de características por tier).

================================================================================
ESTRUCTURA DE LA LICENCIA
================================================================================

Ejemplo de licencia generada:

  license_key: IBOT-D399BB-61BF0FD3F930
  ├─ IBOT = Prefijo de la aplicación
  ├─ D399BB = Hash único del nombre del cliente
  └─ 61BF0FD3F930 = Token aleatorio de seguridad

Campos en Google Sheets:

  license_key        → Identificador único (REQUERIDO)
  active_status      → TRUE o FALSE (controla si funciona)
  expiry_date        → YYYY-MM-DD (fecha de vencimiento)
  customer_name      → Nombre del cliente (para auditoría)
  created_date       → Fecha de creación (para auditoría)
  features           → basic | standard | premium (para futuro)
  max_instances      → Número máximo de instalaciones

================================================================================
FLUJO DE VALIDACION EN TIEMPO REAL
================================================================================

Cuando cliente ejecuta "streamlit run src/app.py":

1. INTENTA CONECTAR A ENDPOINT (si hay internet)
   ├─ Descarga JSON de licencias desde Google Apps Script
   ├─ Busca license_key del cliente
   ├─ Valida active_status = TRUE
   ├─ Valida expiry_date > hoy
   ├─ RESULTADO: Licencia válida ✓
   └─ Guarda caché local

2. SI NO HAY INTERNET
   ├─ Intenta leer caché local
   ├─ Si caché existe y <7 días: Licencia válida ✓
   └─ Si caché vencido: Error (usuario offline >7 días)

RESULTADO FINAL:
  ✓ Licencia válida → Aplicación inicia
  ✗ Licencia inválida → Mensaje de error + st.stop()

================================================================================
SEGURIDAD
================================================================================

PROTECCIONES:
  ✓ Google Sheet PRIVADA (no se puede acceder directamente)
  ✓ Endpoint solo devuelve datos (no permite crear/editar)
  ✓ Hash único + token aleatorio en cada licencia
  ✓ Validación en servidor (no local)
  ✓ Expiración automática

COMO NO SE PUEDE HACKEAR:
  ✗ No puedes ver todas las licencias (solo endpoint privado)
  ✗ No puedes crear licencias falsas (son únicas con hash)
  ✗ No puedes editar Google Sheet (es privada)
  ✗ No puedes falsificar validación (necesita internet o caché)

================================================================================
ESTADISTICAS Y REPORTES
================================================================================

Google Sheets proporciona automáticamente:

1. CLIENTES ACTIVOS
   └─ Contar filas con active_status = TRUE

2. LICENCIAS POR VENCER
   └─ Filtrar expiry_date < [fecha próximas 30 días]

3. INGRESOS RECURRENTES
   └─ (count de STANDARD + PREMIUM) * precio_mensuario

4. TASA DE RENOVACIÓN
   └─ Clientes que renovaron / Clientes vencidos

5. AUDITORÍA
   └─ Ver created_date y customer_name para tracking

Puedes agregar formulas en Google Sheets para automatizar reportes.

================================================================================
COMO REALIZAR UN CAMBIO DE LICENCIA
================================================================================

Si cliente solicita renovación / upgrade / downgrade:

1. Abre Google Sheets
2. Encuentra la fila del cliente
3. Modifica:
   - expiry_date → Nueva fecha
   - features → Nuevo tier
   - max_instances → Nuevas instancias
4. Guarda
5. Avisa al cliente que está activo

El cliente no necesita cambiar nada en su lado (se valida automáticamente).

================================================================================
COMO DESACTIVAR UNA LICENCIA
================================================================================

Si cliente no paga o viola términos:

1. Abre Google Sheets
2. Encuentra la fila del cliente
3. Cambia: active_status → FALSE
4. Guarda
5. La próxima vez que cliente ejecute la app (con internet):
   └─ Verá: "Licencia inactiva"
   └─ st.stop() detiene la ejecución

Efecto inmediato (1-2 minutos después de cambiar).

================================================================================
TROUBLESHOOTING
================================================================================

Cliente ve: "Licencia no encontrada"
  └─ Verificar que license_key está en Google Sheets
  └─ Verificar que está escrito exactamente igual (sin espacios)

Cliente ve: "Licencia inactiva"
  └─ Verificar active_status = TRUE en Google Sheets
  └─ Posiblemente el cliente no pagó

Cliente ve: "Licencia expirada"
  └─ Verificar expiry_date > hoy
  └─ Renovar cambando la fecha en Google Sheets

Cliente está offline >7 días
  └─ Necesita internet para revalidar
  └─ El caché local dura máximo 7 días

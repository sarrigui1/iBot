================================================================================
CONFIGURAR API KEY DE ANTHROPIC - GUIA COMPLETA
================================================================================
Fecha: 2026-04-05
Versión: 1.0

================================================================================
RESUMEN
================================================================================

iBot Trading requiere una API key válida de Anthropic (Claude API) para:
- Análisis de mercado con IA
- Decisiones de trading inteligentes
- Análisis SMC automático

Sin la API key, el sistema NO puede hacer análisis de IA.

================================================================================
PASO 1: CREAR CUENTA EN ANTHROPIC
================================================================================

1. Ve a: https://console.anthropic.com/
   (o https://claude.ai si no tienes cuenta aún)

2. Haz clic en "Sign up"

3. Completa el formulario:
   - Email
   - Contraseña
   - Información personal

4. Verifica tu email

5. ¡Cuenta creada!

================================================================================
PASO 2: GENERAR API KEY
================================================================================

1. Inicia sesión en: https://console.anthropic.com/

2. En el menú de la izquierda, haz clic en: "API Keys"
   (O ve directamente a: https://console.anthropic.com/keys)

3. Haz clic en el botón: "Create Key"

4. Elige opciones:
   - Name: "iBot Trading" (o cualquier nombre descriptivo)
   - Expiration: "No expiration" (recomendado) o elige una fecha

5. Haz clic en: "Create"

6. ¡IMPORTANTE! Copia la API key completa
   - Verás algo como: sk-ant-v0-xxxxxxxxxxxxxxxxxxxx
   - COPIAR COMPLETO (es muy largo)
   - NO compartir con nadie
   - NO publicar en GitHub

7. Guarda en un lugar seguro (documento privado, gestor de contraseñas, etc.)

================================================================================
PASO 3: CONFIGURAR EN IBOT
================================================================================

1. Abre el archivo: config/config.ini

2. Busca la sección: [API_KEYS]

3. Reemplaza:
   ANTHROPIC_API_KEY = tu_api_key_aqui

   Con:
   ANTHROPIC_API_KEY = sk-ant-v0-xxxxxxxxxxxxxxxxxxxxx

   Ejemplo real (FALSO - no uses este):
   ANTHROPIC_API_KEY = sk-ant-v0-abcd1234efgh5678ijkl9012mnop3456

4. Guarda el archivo

5. Cierra y reabre launcher.bat

6. ¡Listo! Ahora iBot puede usar IA

================================================================================
VERIFICAR QUE FUNCIONA
================================================================================

Después de configurar la API key:

1. Abre launcher.bat

2. Espera a que Streamlit inicie (verás: "Local URL: http://localhost:8501")

3. Abre en navegador: http://localhost:8501

4. Activa la opción "AI" en la aplicación

5. Si funciona:
   ✓ Verás análisis de trading
   ✓ El log dirá: "API key validada"
   ✓ Decisiones de IA se mostrarán

6. Si falla:
   ✗ Verás error 401 o "authentication error"
   ✗ Revisa que la API key sea exacta (sin espacios extras)
   ✗ Verifica en https://console.anthropic.com/ que la clave esté activa

================================================================================
ERRORES COMUNES Y SOLUCIONES
================================================================================

Error: "invalid x-api-key"
  └─ Causa: API key es inválida, vencida o mal copiada
  └─ Solución:
     1. Copia la API key de nuevo desde console.anthropic.com
     2. Verifica que NO haya espacios al inicio o final
     3. Asegúrate de haber copiado COMPLETA (es muy larga)
     4. Regenera una nueva si es necesario

Error: "ANTHROPIC_API_KEY no está configurada"
  └─ Causa: No pusiste la API key en config.ini
  └─ Solución:
     1. Abre config/config.ini
     2. En [API_KEYS], agrega la API key real
     3. NO uses "tu_api_key_aqui" (eso es solo un placeholder)

Error: "authentication_error"
  └─ Causa: Credenciales rechazadas por Anthropic
  └─ Solución:
     1. Verifica en https://console.anthropic.com/
     2. La clave debe estar ACTIVA (no desactivada)
     3. Si es muy vieja, prueba regenerarla
     4. Verifica que NO haya límites de cuota excedidos

Error: "connection timeout" o "HTTP 529"
  └─ Causa: Servidor de Anthropic congestionado o tu conexión es lenta
  └─ Solución:
     1. Espera unos minutos y reintentar
     2. Verifica tu conexión a internet
     3. Los reintentos son automáticos (2 intentos por defecto)

Error: "Invalid model ID 'claude-haiku-...'"
  └─ Causa: El modelo no existe o tu plan no lo permite
  └─ Solución:
     1. Verifica en https://console.anthropic.com/settings/usage
     2. iBot usa: claude-haiku-4-5-20251001 (económico)
     3. Si no tienes acceso, contacta a Anthropic

SEGURIDAD - PROTEGER TU API KEY
================================================================================

✓ HACER:
  - Guardar en lugar seguro (no en email)
  - Usar gestor de contraseñas
  - Regenerar si crees que fue comprometida
  - Revisar uso regularmente

✗ NO HACER:

   - Ve a "Billing Limits"
   - Fija un límite máximo mensual
   - iBot se detendrá si lo excede

================================================================================
PREGUNTAS FRECUENTES
================================================================================

¿Puedo usar una API key gratuita?
  → Sí, pero tiene límites (15K tokens/día).
  → Para uso en vivo, considera un plan de pago.

¿Es seguro poner la API key en config.ini?
  → Sí, si el archivo está protegido.
  → Asegúrate de NO compartir config.ini.
  → En producción, considera variables de entorno.

¿Qué pasa si se agota el crédito?
  → Las llamadas fallarán con error 429 o 402.
  → iBot mostrará: "Billing quota exceeded".
  → Actualiza tu plan o espera al próximo ciclo.

¿Necesito IA para que funcione iBot?
  → No, puedes desactivar la opción en la UI.
  → Sin IA, funciona como analizador técnico.
  → Con IA, agrega decisiones automáticas.

¿Puedo usar otra IA (OpenAI, etc.)?
  → Actualmente no, iBot está diseñado para Anthropic.
  → Próximas versiones podrían soportar múltiples APIs.
